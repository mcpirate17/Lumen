## Known Issues for Tomorrow

### Critical: Response Quality
- **Models say "I don't have market data" despite data being injected** — the 0.8B model sometimes ignores the [LIVE DATA] block in the prompt. Need to tune the prompt format or use structured extraction instead of free generation.
- **Fallback path has no data** — if SSE stream fails, the JS error handler calls Ollama directly WITHOUT domain data, so the model genuinely has no data. Need to make the fallback also inject cached data.
- **Self-certainty check adds "I'm not 100% sure" prefix** — too cautious, makes Lumen seem unsure even when the data is right there. Disable or tune the threshold.

### Response Latency
- API side is fast: 0.5s greeting, 2.5s finance (speculative), 1.5s sports
- Perceived slowness is TTS synthesis (old Kokoro server) + prompt eval time with history
- 4B model takes 4s just on prompt eval when history + domain data is included
- Need to reduce prompt size or move to MLX (expected 7x speedup)

### Voice
- Voice swapped to af_bella (female) — working
- Old Kokoro server is slow at synthesis
- Need to switch to new tts_server.py or install kokoro-onnx for faster TTS

### TinyBERT Not Loading
- `transformers` 5.3.0 broke the `pipeline` import
- Need to pin transformers version or fix import

### MLX Migration Blocked
- Qwen3.5 models are gated on HuggingFace
- Need: `huggingface-cli login`
- Current Ollama: 52 tok/s (0.8B), 22 tok/s (4B)
- Expected MLX: 525 tok/s, 159 tok/s (10x / 7x improvement)
